# UI 画像スキン — 作成ルール

`pygame_ui` 用の PNG スキンを、自分で作るときの手順とデザイン上のルールです。  
コードの挙動は `pygame_ui/skin_draw.py` に合わせて書いています。

---

## 1. スキンとは何か

1 つのフォルダに **`skin.json`（目録）** と **PNG（見た目）** を置きます。

```text
my_skin/
  skin.json
  button_idle.png
  button_hover.png
  ...
```

ゲーム側では次のように読み込みます。

```python
from pygame_ui import UITheme

theme = UITheme.from_skin_dir("path/to/my_skin")
# または組み込み: UITheme.with_skin("pixel")
```

**一部の PNG だけ用意しても動きます。** 無い部位はテーマのベクター描画（色・ベベル枠）に自動で戻ります。

---

## 2. 描画の 2 種類（ここが最重要）

| 方式 | `skin.json` の書き方 | 画面上の動き | 向いているもの |
|------|----------------------|--------------|----------------|
| **伸縮（ストレッチ）** | `images` にキーを書く | PNG 全体をウィジェットの矩形いっぱいに **縦横とも引き伸ばす** | ボタン、チェック、スライダーつまみ、メニュー行 |
| **9-slice（ナインスライス）** | `nine_slice` にマージンを書く | 四隅は固定、辺は一方向だけ伸ばす、中央は両方向伸ばす | パネル、メニュー背景、細長いトラック |

### 9-slice のイメージ

元画像を **左・上・右・下** のピクセル数で 9 分割します（`left` / `top` / `right` / `bottom`）。

```text
+-------+-----------+-------+
| 固定  |  上辺     | 固定  |  ← top
| 左上  |  (横伸)   | 右上  |
+-------+-----------+-------+
| 左辺  |  中央     | 右辺  |  ← 左右・中央はウィジェットサイズに合わせて伸縮
| (縦)  | (縦横)    | (縦)  |
+-------+-----------+-------+
| 左下  |  下辺     | 右下  |  ← bottom
+-------+-----------+-------+
```

**マージンの決め方（実務ルール）**

- 角の装飾（枠線・丸み・影）が **伸びてほしくない範囲** を `left` / `top` / `right` / `bottom` に入れる。
- 中央は **単色やごく弱い模様** にすると、大きいパネルでも破綻しにくい。
- 必ず **`left + right < 画像の幅`**、**`top + bottom < 画像の高さ`**（最低でも各辺 1px は中央に残す）。
- ウィジェットが小さすぎるときは、画像全体を無理やり縮小するフォールバックがあります（細かい角は潰れます）。

### ストレッチの注意

ボタンなどは **ラベル長に応じて幅が変わります**。  
64×24 の素材を 200×40 に伸ばすと、枠線も太く見えます。

- 対策 A: 9-slice 用の `panel.png` を流用しない（ボタンは `images` のまま、シンプルな塗り＋上下対称の枠）。
- 対策 B: デザイン時点で「よく使う最大幅」くらいの比率で作る。
- 対策 C: 将来、ボタンも 9-slice 対応を足す余地あり（現状は伸縮のみ）。

## 3. フォルダと `skin.json`

### 3.1 最小構成

```json
{
  "name": "my_skin",
  "images": {},
  "nine_slice": {}
}
```

- **`name`**: 表示用。フォルダ名と揃えると分かりやすい。
- **`images`**: キー → **ファイル名**（同じフォルダ内の相対パス）。
- **`nine_slice`**: キー → オブジェクト（下記）。

### 3.2 `images` の例

```json
"images": {
  "button_idle": "button_idle.png",
  "button_hover": "button_hover.png",
  "button_pressed": "button_pressed.png"
}
```

### 3.3 `nine_slice` の例

```json
"nine_slice": {
  "panel": {
    "image": "panel.png",
    "left": 8,
    "top": 8,
    "right": 8,
    "bottom": 8
  }
}
```

- `image` または `file` で PNG を指定（どちらでも可）。
- 数値は **整数ピクセル**。省略時はコード側デフォルト 4。

### 3.4 組み込みスキンの場所

| 種類 | パス |
|------|------|
| 同梱サンプル | `pygame_ui/assets/skins/pixel/` |
| 自作 | 任意（`UITheme.from_skin_dir` に渡す） |

## 4. 画像キー一覧（必須・任意・フォールバック）

### 4.1 ボタン（`images`・伸縮）

| キー | いつ使う | 必須 | 無いとき |
|------|----------|------|----------|
| `button_idle` | 通常 | 推奨 | ベクター |
| `button_hover` | マウスオーバー | 任意 | `button_idle` |
| `button_pressed` | 押下中 | 任意 | `button_idle` |
| `button_disabled` | 無効 | 任意 | `button_idle` |

- 描画矩形: **ボタンの `rect` 全体**（ラベルはその上に文字）。
- サンプルサイズ: **64×24**（`scripts/generate_ui_skin.py`）。

### 4.2 チェックボックス（`images`・伸縮）

| キー | いつ使う | 必須 |
|------|----------|------|
| `checkbox_off` | オフ | 推奨 |
| `checkbox_on` | オン | 推奨 |

- 描画矩形: **18×18 固定**（`checkbox.py` の `BOX_SIZE`）。ラベルは右側、別途テキスト。
- スキン使用時は **ベクターのチェックマークは描かない**（ON/OFF は画像だけで表現する想定）。
- サンプル: **18×18**。

### 4.3 スライダー

| キー | 方式 | 描画先 | 備考 |
|------|------|--------|------|
| `slider_track` | `nine_slice` または `images` | トラック矩形 | 高さはウィジェットから **おおよそ 4〜8px** |
| `slider_fill` | `images` | 値に応じた左側の矩形 | 0 のときは描かない |
| `slider_thumb` | `images` | つまみ矩形 | 幅 **14px** 固定、高さ ≒ `rect.h - 4` |

- トラック幅 = スライダー全体の幅（可変）。
- `nine_slice.slider_track` を使うと、横に長いトラックでも端の模様が保たれやすい。
- サンプル: トラック **48×10**、つまみ **14×22**。

### 4.4 パネル

| キー | 方式 | 備考 |
|------|------|------|
| `nine_slice.panel` | 9-slice | **推奨** |
| `images.panel` | 伸縮 | `nine_slice.panel` が無いときのみ |

- 描画矩形: パネルの `rect`（可変）。
- サンプル `panel.png`: **48×48**、マージン **8** 四方。

### 4.5 コンテキストメニュー

| キー | 方式 | 備考 |
|------|------|------|
| `nine_slice.menu` | 9-slice | 無ければ `nine_slice.panel` |
| `images.menu_bg` | 伸縮 | 上記が無いとき。さらに無ければ `images.panel` |
| `images.menu_row_hover` | 伸縮 | ホバー行のみ。行の高さ **28px**（`ROW_H`） |

- メニュー全体のサイズは項目数・文字幅で毎回変わる → **背景は 9-slice 推奨**。
- 行ハイライトは **幅 = メニュー幅** に伸縮。サンプル **120×24**。

## 5. PNG ファイルの共通ルール

| 項目 | ルール |
|------|--------|
| 形式 | **PNG**（非圧縮・透過どちらでも可） |
| 透過 | **RGBA 推奨**（角丸・影はアルファで） |
| 色深度 | 8bit / 24+8bit どちらでも可 |
| サイズ | 2 の冪にする必要は **なし** |
| 命名 | `skin.json` のファイル名と **完全一致**（大文字小文字に注意） |
| 解像度 | ドット絵なら **1px = 1論理px** で作り、拡大はゲーム側の画面スケールに任せる |

**やってはいけないこと**

- `skin.json` だけ置いて PNG を忘れる（そのキーはスキップされ、ベクターに戻る）。
- 9-slice のマージンを画像より大きくする（全体縮小フォールバックになり、意図とずれる）。
- チェック ON で「小さな ✓ だけ」描いて、18×18 に伸ばす（✓ も太くなる）。ON/OFF で **別画像** を使う。

**推奨ワークフロー**

1. 同梱 `pixel/` をコピーして `my_skin/` を作る。  
2. 1 ウィジェットずつ差し替え（例: ボタン 3 枚だけ）→ デモで確認。  
3. パネルは 9-slice マージンをエディタのガイド線で合わせる。  
4. 完成したら `UITheme.from_skin_dir` で読み込み。

再生成用サンプル（参考用 PNG）:

```bash
python scripts/generate_ui_skin.py
```

## 6. ウィジェットごとのサイズ目安（サンプル `pixel`）

実際の画面上では **矩形に合わせて伸縮** されます。素材は「見本サイズ」として作れば十分です。

| ファイル（例） | サンプルピクセル | 用途 |
|----------------|------------------|------|
| `button_*.png` | 64 × 24 | ボタン 3 状態 |
| `checkbox_*.png` | 18 × 18 | チェック |
| `slider_track.png` | 48 × 10 | トラック（9-slice マージン 4,2,4,2） |
| `slider_fill.png` | 48 × 10 | 塗り部分（左から伸縮） |
| `slider_thumb.png` | 14 × 22 | つまみ |
| `panel.png` | 48 × 48 | パネル・メニュー背景 |
| `menu_row_hover.png` | 120 × 24 | 行ハイライト |

## 7. デザインのコツ（ピクセルアート向け）

1. **パレットを固定**（16〜32 色）すると、idle / hover / pressed の差が揃う。  
2. **光源を一方向に統一**（例: 左上ハイライト、右下シャドウ）。サンプルはベベル枠。  
3. **ボタン 3 枚**は同じキャンバスサイズにそろえる（切り替え時のジャンプ防止）。  
4. **パネル 9-slice** の中央 1px は、タイルとして繰り返しても目立たない色・模様にする。  
5. **日本語ラベル**はスキンに含めない（フォントは `UITheme.font_path`）。画像は枠だけ。

## 8. 動作確認

```bash
# ユニットテスト（ヘッドレスでスキン読み込み）
python -m pytest tests/test_pygame_ui.py::TestSkin -q

# 目視（T キーで default → pixel_art → skin:pixel）
python -m pygame_ui.demo
```

自作スキンをデモに載せる例（`demo.py` を一時的に編集）:

```python
themes = [
    UITheme.from_defaults(),
    UITheme.from_skin_dir(r"C:\path\to\my_skin"),
]
```

## 9. チェックリスト（公開前）

- [ ] `skin.json` の JSON がパースできる（カンマ・引用符）
- [ ] 列挙した PNG がすべてフォルダに存在する
- [ ] 9-slice の `left/top/right/bottom` が画像内に収まる
- [ ] ボタン hover / pressed が idle と同じ外寸
- [ ] チェック on/off が 18×18 想定で意図どおり
- [ ] 細いスライダー（高さ 4px）でもトラックが潰れないかデモで確認
- [ ] 大きいパネルで 9-slice の中央が繰り返し目立たないか確認
