## 1. 作品概要

開発中のゲームは、**蟻コロニー運営**をテーマにした**大規模シミュレーション型タワーディフェンス**である。

プレイヤーは人型に近く擬人化された**女王蟻**を中心としたコロニーを運営し、生態系の中で資源を確保しながら勢力を拡大していく。

### コア体験

> **大量の自律 AI 個体が作り出す蟻社会を観察する楽しさ**

タワーディフェンスはその社会を成立させる要素の**一つ**。単なる防衛ゲームではなく、**生態系シミュレーション・コロニー運営・AI 観察・文明発展**を融合させる。

### プレイヤーの立場

プレイヤーは個体を直接操作するのではなく、**環境・方針・限定的支援**を与えながらコロニーを導き、**観察する**立場に近い。最終的には数千〜数万規模の個体が同時に活動する世界を目指す（長期）。

---

## 2. ゲーム進行 — 3 フェーズ

ゲームは主に次の 3 フェーズで進行する。フェーズの**意味付け・切替・目標管理**は **Game 層**の責務（Sim は個体の物理・AI 実行のみ）。

### 2.1 開発フェーズ

働き蟻が資源収集・巣の拡張を行うフェーズ。

| 要素 | 内容 |
|------|------|
| 資源 | 食料・資材の収集と備蓄管理 |
| 繁殖 | 女王蟻の産卵 → 働き蟻・兵隊蟻の増加 |
| 整備 | 防衛施設・巣構造の整備（次の戦闘へ備える） |
| 将来 | 文明技術の研究、高度施設建設 |

### 2.2 防衛フェーズ

敵が**ウェーブ形式**で襲来する戦闘フェーズ。

| 要素 | 内容 |
|------|------|
| 戦闘 | 兵隊蟻は基本 **AI 自律行動**。プレイヤーは全体方針・限定的支援 |
| 規模 | 大量個体の群れ戦が見どころ |
| 将来 | 大型生物・敵勢力とのレイド戦 |

### 2.3 ストーリーフェーズ

開発・防衛の**合間**に挿入される短時間イベントパート。

| 目的 | 例 |
|------|-----|
| 世界観・状況説明 | オープニング、季節変化 |
| 次の目標提示 | ウェーブクリア後、危機発生時 |
| 新要素解放演出 | 文明レベル上昇、新システム |
| 愛着形成 | 女王蟻・ネームド個体 |
| 歴史の記録 | プレイヤーごとの「コロニーの歴史」 |

**演出方針**: 処理負荷を抑え、**立ち絵・簡易アニメ・テキスト会話**を中心とする。長時間の会話シーンは避ける。

**ナラティブ**: 女王蟻中心。一部の兵隊蟻・働き蟻がネームド個体として登場しうる。

**動的ストーリー**: 固定シナリオに加え、プレイ中の出来事を反映する。

- 突然変異個体の誕生
- 大飢饉を乗り越えた
- 巨大生物を撃退した
- 他コロニーとの戦争に勝利した
- 新文明技術の獲得

### ストーリー発生タイミング（想定）

- ゲーム開始（オープニング）
- ウェーブクリア後
- レイドボス討伐後
- 文明レベル上昇時
- 季節変化時
- 新勢力・新生物との遭遇時
- 重要な突然変異個体誕生時
- コロニー存続の危機発生時

## 3. 実装ロードマップ

すべてを一度に導入しない。**プロトタイプ → 段階拡張 → どこかで「完成」区切り**を想定する。

### 3.1 初期（プロトタイプ核）

軽量システムで **「大量 AI 個体を眺めるだけでも面白い」** を成立させる。

| 要素 | 概要 |
|------|------|
| コロニー運営 | 女王蟻・働き蟻・兵隊蟻 |
| 資源 | 収集・備蓄 |
| 巣 | 拡張・簡易建築 |
| 防衛 | ウェーブ形式 |
| 群体 | 大量個体の行動 |
| AI | 敵味方の基本 AI |
| 突然変異 | 自然発生・一騎当千・予想外展開 |
| 観察 | 俯瞰 + **特定個体追跡**（強個体・長寿個体への愛着） |

**プロトタイプ完成の目安（Game 層）**:  
開発 ↔ 防衛（ウェーブ 1 本以上）↔ ストーリー（短尺）の**ループが 1 周回る**こと。

### 3.2 中期

| 要素 | 概要 |
|------|------|
| フェロモン | 餌・危険・集合・探索など。AI 間情報共有。プレイヤーによる間接誘導 |
| 遺伝 | 突然変異を系統管理（高火力・高速・重装甲・大型・高知能等） |
| 女王成長 | 産卵効率・新種解放・特殊フェロモン・研究能力 |
| 生態系強化 | 餌生物・植物・他昆虫の自律増減 |
| 季節 | 春夏秋冬。冬越し備蓄が戦略要素 |

### 3.3 長期

| 要素 | 概要 |
|------|------|
| 勢力争い | 他コロニー・昆虫勢力。偵察・領土・資源争奪・巣侵攻。動的勢力図 |
| 巨大生物 | カマキリ・ムカデ・クモ・カエル・鳥等。蟻群 vs 巨大生物 |
| 文明発展 | 武器・防壁・罠・自動砲台・機械。生態と文明の融合 |
| 人類接触 | 金属・電池・プラスチック・電子部品等の回収 |
| 超大規模 | 数千〜数万同時個体 |

---

## 4. 開発方針

1. **コア体験優先** — 観察の面白さを最速で成立
2. **段階的複雑化** — フェロモン・遺伝・季節等は後から
3. **完成ラインを設ける** — 全長期要素の完全実装は目標としない
4. **層分離を維持** — Sim / Game / Client の契約（[Layer_Interfaces.md](./Layer_Interfaces.md)）

## 5. コードベースとの対応（Game 層 AI 向け）

境界分離は [REVIEW_Sim_Game_Layer_Independence_Problems.md](./REVIEW_Sim_Game_Layer_Independence_Problems.md) §0 で凍結済み。**新規ゲーム意味は Game 層に書く。**

| 企画要素 | 主な配置 | 現状（2026-06） |
|----------|----------|-----------------|
| コロニー経済・備蓄 | `colony_orchestrator`, `colony_compound` | あり |
| 行動 AI（狩り・戦闘・給餌等） | `src/game/ai/*` | あり |
| イベント解釈・初回フラグ | `game_director`, `game_state` | あり |
| 解禁・進行 | `progression.py`, `config/game/progression.json` | 骨格あり |
| プレイヤー向け API | `client_api.py` | 一部 |
| **フェーズ制御** | `phase_controller`, `wave_director`, `client_api` | **実装済み（最小 1 ウェーブ）** |
| **ウェーブ防衛** | `wave_director`, `config/game/waves.json` | **実装済み（wave_1）** |
| **ストーリー** | `phase_controller` STORY フェーズ + `client_api` | **最小（自動タイマー / ack）** |
| **突然変異（ルール）** | （未）Game 側マネージャ | **未実装** |
| 個体物理・近傍・戦闘事実 | Sim 層 | Sim 担当 |
| 描画・入力・観察 UI | Client / `pygame_ui` | Client 担当 |

### Game → Sim / Client の原則

- **Sim への書き込み**: `SimBridge.execute` のみ
- **Sim からの通知**: `world.events.drain()` → `GameDirector`
- **Client への公開**: `GameController`, `GameState`, `client_api`

## 6. 用語

| 用語 | 意味 |
|------|------|
| affiliation | 勢力 ID（プレイヤーコロニー・ライバル等）。コード上の中立 ID |
| 女王蟻 | プレイヤー焦点。産卵・ストーリー中心 |
| 突然変異 | 通常を大きく上回る能力。自然発生。ネームド化・履歴連携 |
| 観察モード | 俯瞰 + 個体追跡。操作より観察 |
| ウェーブ | 防衛フェーズの敵襲来単位 |
